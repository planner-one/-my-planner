### SC-03: AppContext + Firestore 동기화

- **목적**: 전역 상태와 Firestore 저장 전략 구현
- **주요 흐름**:
  1. `userService.ts`에서 사용자 데이터 로드/저장
  2. `AppContext.tsx`에서 Firestore 필드 전체 상태 관리
  3. `currentDataRef`로 최신 상태 추적
  4. 일반 변경은 1초 debounce 저장
  5. 레이아웃 저장과 로그아웃 전 저장은 즉시 저장
- **완료 기준**: 새로고침 후 데이터 복원, 계정 전환 시 localStorage 정리, pending save가 이전 uid와 계정 메타데이터를 유지

### SC-10: 태블릿/모바일 반응형

- **목적**: 태블릿과 모바일에서 대시보드와 기능 페이지를 사용할 수 있게 검토
- **검토 기준**:
  - 데스크톱, 태블릿 가로/세로, 모바일 폭별 주요 화면 확인
  - 모바일 대시보드 세로 카드형 위젯 구조
  - 하단 탭 대표 메뉴 6개
  - 모바일 상단 전체 메뉴를 통한 모든 페이지 접근
  - 하위 페이지 홈 복귀 버튼
  - 각 페이지 입력 폼, 카드 폭, 테이블/히트맵/캘린더 스크롤
  - 태블릿/모바일 위젯 드래그/리사이즈 지원 여부는 미확정
- **주의사항**:
  - 현재 완료 상태는 코드/빌드 기준입니다. 실제 로그인 세션에서 모바일 390px, 태블릿 세로 768px, 태블릿 가로 1024px 화면을 최종 확인하기 전에는 "실기기 최적화 완료"로 표현하지 않습니다.
  - 모바일 하단 탭은 대표 메뉴 6개만 노출합니다. 전체 페이지 접근은 상단 전체 메뉴 또는 메뉴 위젯으로 확인해야 합니다.
  - 캘린더, 습관 히트맵, 주간표, 출력 미리보기처럼 정보량이 많은 화면은 모바일에서 가로 스크롤 또는 축약 표시가 섞여 있으므로 텍스트 겹침, 버튼 잘림, 입력 폼 폭을 우선 확인합니다.
  - 태블릿/모바일 위젯 드래그/리사이즈는 지원 여부가 미확정입니다. 결정 전까지 지원 기능으로 안내하지 않습니다.
- **완료 메모**: 2026-06-29 기준 주요 페이지에 모바일/태블릿 브레이크포인트를 적용했고, 인앱 브라우저 연결 제한으로 실제 로그인 세션 육안 QA는 사용자 확인 권장 사항으로 남깁니다.

### SC-12: AI 링크 자동 정리 / 유료 기능 후보

- **목적**: 사용자가 링크를 입력하면 AI가 내용을 분석해 신청 관리, 캘린더, 작업 목록, 메모 중 필요한 위치에 자동 정리하도록 설계
- **상태**: 2026-07-01 기준 1차 기반 구현. 실제 웹페이지 본문 분석, 링크 페이지 내부 이미지 자동 수집, 유료 권한, 사용량 제한은 추후 백엔드/API 연결 검토 대상으로 관리
- **구현된 범위**:
  - 전역 "링크 정리" 버튼과 링크 입력 모달
  - URL/사용자 메모 기반 초안 생성 서비스 경계
  - 이미지 포스터 자료 유형, 업로드 이미지 OCR, 포스터 텍스트 기반 날짜/시간/장소 초안 생성
  - 신청 관리, 캘린더, 작업 목록, 메모로 사용자 확인 후 삽입
  - 신청관리 작성 폼의 "링크에서 불러오기" 버튼과 폼 자동 채움
  - `sourceUrl` 원본 링크 보존
- **예상 진입점**:
  - 전역 "링크 정리" 버튼: 사이드바 하단과 상단 페이지 헤더 보조 액션
  - 모바일: 상단 헤더의 링크 정리 버튼
  - 신청관리 페이지: 작성 모달/폼 상단의 "링크에서 불러오기" 버튼
- **예상 흐름**:
  1. 사용자가 링크 입력 모달에 URL을 붙여넣음
  2. AI가 링크 내용, 업로드 이미지 OCR 결과, 이미지 포스터에서 읽은 텍스트를 분석해 제목, 요약, 일정, 신청 마감일, 카테고리, 상태, 장소/온라인 여부, 원본 URL을 추출
  3. 앱이 신청 일정, 캘린더, 작업 목록, 메모 중 삽입 후보를 제안
  4. 사용자가 미리보기에서 수정/확인
  5. 확인 후 AppContext/Firebase 저장 흐름에 맞춰 각 데이터에 삽입
- **주의사항**:
  - 외부 페이지 크롤링, CORS, 로그인 필요한 페이지, robots/약관 준수, API 키 보호 때문에 프론트엔드 단독 구현은 제한적입니다.
  - 유료 기능으로 운영하려면 사용자 권한, 사용량 제한, 실패/환불 정책, 개인정보 처리방침, 원본 링크 보관 정책이 필요합니다.
  - AI 추출 결과는 오입력 가능성이 있으므로 자동 저장 전 사용자 확인을 필수로 둡니다.
  - 현재 1차 구현은 업로드 이미지 파일 OCR을 지원하지만, 실제 웹페이지 본문이나 링크 페이지 내부 이미지를 자동 수집하지는 않습니다.
  - Google Form처럼 로그인, 쿠키 허용, 접근 권한이 필요한 링크는 현재 앱과 Codex 모두 실제 폼 내용을 읽지 못할 수 있으므로 본문 분석 불가 안내와 수동 보정 흐름을 제공합니다.
  - 공개 공고/행사/교육/채용 페이지처럼 자동 정리에 적합한 링크 기준과 제한 사항은 `LINK_IMPORT_GUIDE.md`를 기준으로 안내합니다.
  - 이미지 포스터형 게시글은 현재 이미지 파일 업로드 OCR 또는 포스터 텍스트 붙여넣기 기반으로 정리하고, 링크 페이지 내부 이미지 자동 수집/고정밀 이미지 분석은 추후 유료 기능 후보로 관리합니다.
  - 브라우저 OCR은 첫 실행 시 언어 데이터 로딩 시간이 걸릴 수 있고, 실패하거나 오인식된 텍스트는 사용자가 미리보기에서 보정합니다.
