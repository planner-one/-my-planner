### SC-01: 프로젝트 초기 셋업

- **목적**: Vite + React + TypeScript + Firebase 연결, 폴더 구조 생성
- **완료 기준**: `npm run dev`, `npm run build` 정상 실행, Firebase Hosting 설정 확인

---

### SC-02: Google 로그인 / 로그아웃

- **목적**: Firebase Auth를 통한 Google 소셜 로그인
- **완료 기준**: 로그인 후 대시보드 진입, 새로고침 후 세션 유지, 로그아웃 전 저장

---

### SC-03: AppContext + Firestore 동기화

- **목적**: 전역 상태와 Firestore 저장 전략 구현
- **주요 흐름**:
  1. `userService.ts`에서 사용자 데이터 로드/저장
  2. `AppContext.tsx`에서 Firestore 필드 전체 상태 관리
  3. `currentDataRef`로 최신 상태 추적
  4. 일반 변경은 1초 debounce 저장
  5. 레이아웃 저장과 로그아웃 전 저장은 즉시 저장
- **완료 기준**: 새로고침 후 데이터 복원, 계정 전환 시 localStorage 정리

---

### SC-04: 앱 라우팅 + 네비게이션

- **목적**: `RouterContext` 기반 페이지 전환, 사이드바/하단 탭/메뉴 위젯 연결
- **현재 page id**:
  - `dashboard`, `tasks`, `todos`, `habits`, `goals`, `projects`, `career`
  - `weekly`, `daily`, `notes`, `journal`, `profile`, `inquiries`, `print`
- **완료 기준**: 사이드바의 모든 페이지가 실제 컴포넌트로 연결되고, 하위 페이지에서 홈 복귀 버튼이 제공됨

---

### SC-05 / SC-06: 대시보드 뷰 + 편집 모드

- **목적**: 위젯 그리드 표시와 편집
- **주요 흐름**:
  1. DashboardEditor 진입
  2. react-grid-layout 1.4.4 렌더
  3. 드래그/리사이즈 완료 시 상태 업데이트
  4. 위젯 추가 패널 열기/닫기
  5. 위젯 삭제
  6. 저장 버튼으로 즉시 저장
- **완료 기준**: 레이아웃 저장 후 새로고침해도 유지

---

### SC-07: 위젯 구현/검토

- **목적**: 대시보드 위젯 14종 구현과 리사이즈 반응형 검토
- **현재 등록 위젯**:
  1. 현재 시각
  2. 빠른 메모
  3. 캘린더
  4. 오늘 할 일
  5. 일일 루틴
  6. 집중 타이머
  7. 카운터
  8. 생산성 추이
  9. 목표
  10. 메뉴
  11. 하루 마무리
  12. 예정된 작업
  13. 날씨
  14. 저널 알림
- **정리된 위젯**:
  - 최우선 목표: 목표 위젯의 오늘 집중 섹션으로 통합
  - 빠른 추가: 오늘 할 일 위젯과 중복되어 삭제
- **완료 기준**: 각 위젯이 AppContext 데이터와 연동되고 크기별 주요 UI가 잘리지 않음

### SC-08: 사이드바 기능 페이지 구현/검토

- **목적**: 사이드바에 노출되는 기능 페이지를 구현하고, 데이터 저장/연동/반응형을 검토
- **전제 조건**: SC-07 완료

| 순서 | page id | 페이지 | 상태 | 다음 액션 |
|------|---------|--------|------|-----------|
| 1 | `habits` | 습관 | 구현됨 | 화면/저장 재검토 |
| 2 | `tasks` | 작업 관리 | 구현됨 | 화면/저장 재검토 |
| 3 | `todos` | 오늘 할 일 | 구현됨 | 화면/저장 재검토 |
| 4 | `goals` | 목표 | 구현됨 | 화면/저장 재검토 |
| 5 | `projects` | 프로젝트 | 스텁 | 구현 필요 |
| 6 | `career` | 신청 관리 | 구현됨 | 화면/저장 재검토 |
| 7 | `weekly` | 주간 | 스텁 | 구현 필요 |
| 8 | `daily` | 일일 | 스텁 | 구현 필요 |
| 9 | `notes` | 노트 | 구현됨 | 화면/저장 재검토 |
| 10 | `journal` | 저널 | 스텁 | 구현 필요 |
| 11 | `profile` | 프로필 | 스텁 | 구현 필요 |
| 12 | `inquiries` | 문의 | 구현됨 | 권한/답변 재검토 |
| 13 | `print` | 플래너 출력 | 안내 페이지 구현됨 | 출력 기능 범위 확인 |

**남은 구현 순서**: `projects` → `weekly` → `daily` → `journal` → `profile`

**주의**: `src/widgets/JournalWidget.tsx`는 외부 피드 기반 저널 알림 위젯이고, `src/pages/Journal.tsx`는 사용자가 직접 작성하는 저널/일기 페이지입니다.

### SC-09: 테마 정리

- **목적**: `light`, `dark`, `coral`, `blue` 4개 테마를 최종 정리
- **검토 기준**:
  - CSS 변수 일관성
  - 사이드바 테마 선택 저장
  - 페이지/위젯에서 텍스트 대비와 배지 색상 확인

---

### SC-10: 태블릿/모바일 반응형

- **목적**: 태블릿과 모바일에서 대시보드와 기능 페이지를 사용할 수 있게 검토
- **검토 기준**:
  - 데스크톱, 태블릿 가로/세로, 모바일 폭별 주요 화면 확인
  - 모바일 대시보드 세로 카드형 위젯 구조
  - 하단 탭 대표 메뉴 6개
  - 하위 페이지 홈 복귀 버튼
  - 각 페이지 입력 폼, 카드 폭, 테이블/히트맵/캘린더 스크롤
  - 태블릿/모바일 위젯 드래그/리사이즈 지원 여부는 미확정
