### SC-01: 프로젝트 초기 셋업
- **목적**: Vite + React + Firebase 연결, 폴더 구조 생성
- **주요 흐름**:
  1. `npm create vite` (react-ts 또는 react)
  2. 패키지 버전 고정 설치
  3. `src/` 폴더 구조 생성
  4. `config/firebase.js` 환경변수 연결
  5. Firebase Hosting 초기화
- **완료 기준**: `npm run dev` 정상 실행, Firebase 콘솔 연결 확인

### SC-03: AppContext + Firestore 동기화
- **목적**: 전역 상태 + 3가지 저장 전략 구현
- **전제 조건**: SC-02 완료
- **주요 흐름**:
  1. `userService.js` — loadUserData / saveUserData
  2. AppContext 전역 state 선언 (Firestore 필드 전체)
  3. currentDataRef 패턴 구현
  4. 1초 debounce 저장 effect
  5. saveWithOverrides (즉시 저장)
  6. saveNow (로그아웃 전 즉시 저장 + 3초 타임아웃)
  7. 계정 전환 시 localStorage 초기화
- **완료 기준**: 새로고침 후 데이터 복원 확인

### SC-07: 위젯 14종 구현
- **목적**: 각 위젯 컴포넌트 구현
- **전제 조건**: SC-05/06 완료
- **구현 순서** (의존성 낮은 것 → 높은 것):
  1. ClockWidget (독립)
  2. MemoWidget (독립)
  3. CounterWidget (독립)
  4. PomodoroWidget (독립)
  5. TodoWidget (todos 상태)
  6. HabitWidget (habits 상태)
  7. GoalWidget / TopGoalWidget (goals 상태)
  8. QuickAddWidget (todos 상태)
  9. ScheduledTaskWidget (scheduledTasks 상태)
  10. ReviewWidget (review 상태)
  11. MenuWidget (라우팅)
  12. ChartWidget (chartHistory)
  13. WeatherWidget (외부 API)
- **완료 기준**: 각 위젯이 AppContext 데이터와 연동, 리사이즈 반응형. 캘린더는 긴 일정이 있어도 7열 비율을 유지하고 축소 상태에서도 글씨를 읽을 수 있어야 함
- **빠른 메모 검토 기준**:
  - 위젯에서 짧은 메모를 여러 개 즉시 추가하고 최근 미처리 항목 확인
  - 기존 단일 문자열 메모는 최초 로드 시 개별 빠른 메모 항목으로 이전
  - 노트 페이지에서 작성 날짜별로 묶어 수정·삭제·보관·복원
  - 빠른 메모를 오늘 Todo 또는 제목·본문을 가진 정식 노트로 전환하고 처리 이력 유지
- **일일 루틴 검토 기준**:
  - 최초 사용자는 기본 루틴 5개를 제공받고 이후 자유롭게 추가·수정·삭제·순서 변경
  - 루틴 이름을 변경하거나 현재 목록에서 삭제해도 기존 날짜별 기록은 유지
  - 오늘 체크 결과는 위젯과 습관 페이지에 즉시 반영되고 Firebase에 저장
  - 루틴별 반복 요일을 설정하고 오늘 해당되는 루틴만 위젯의 완료율에 포함
  - 위젯에서 오늘 기록을 저장하며, 저장 후 루틴·체크 변경 시 같은 날짜 기록을 최신 상태로 갱신
  - 습관 페이지에서 최근 1년 전체·개별 루틴 활동을 GitHub 활동 그래프 형태로 조회
- **날씨 위젯 검토 기준**:
  - 최초 사용 또는 자동 위치 모드에서는 브라우저 위치 권한 승인 후 현재 격자 날씨를 즉시 적용·저장
  - 위치 조회 실패 시 임의 지역으로 대체하지 않고 권한 허용 또는 수동 지역 선택 안내
  - 세로 4칸까지 축소 가능
  - 축소 시 지역 선택 영역과 주간 카드가 잘리지 않고 간격·글씨·아이콘이 함께 조정
  - 날짜 카드를 클릭하면 예보 상세 모달 표시
  - 상세 모달에서 현재 좌표 또는 선택 지역 기준의 읽기 쉬운 실시간 날씨 페이지를 새 탭으로 열 수 있음
- **Todo 연동 기준**:
  - TodoWidget과 TodoPage는 동일한 `todos` 상태 사용
  - 오늘 진행률은 TodoPage와 ChartWidget에서만 표시
  - 지난 날짜 결과는 자동 저장
  - 오늘 결과는 TodoPage에서 수동 저장 및 갱신 가능
  - 날짜별 결과와 당시 항목 목록은 Firestore에 보존
  - 저장된 기록의 항목 완료 상태와 메모를 수동 보정 가능
  - 기록에 누락된 항목을 추가하거나 잘못된 항목을 제거 가능
  - 보정 후 완료율을 재계산하고 보정 시각·변경 내역을 누적 보존
  - 잘못 생성된 Todo는 기록 보정에서 개별 삭제하고 삭제 이력을 보존
  - 모든 항목이 제거된 기록과 직접 삭제한 기록은 휴지통으로 이동
  - 휴지통 기록은 복원 또는 영구 삭제 가능하며 자동 저장으로 재생성되지 않음

### SC-08: 페이지 11개 구현
- **목적**: 각 기능 페이지 구현
- **전제 조건**: SC-07 완료 (상태 모델 확정 후)
- **구현 순서**:
  1. HabitTracker (히트맵)
  2. TaskTracker (테이블)
  3. Goals (단계 체크리스트)
  4. Projects (카드)
  5. WeeklyPlanner (요일 카드)
  6. DailyPlanner (시간 블록)
  7. Notes (마크다운 노트)
  8. Journal (일기)
  9. ProfilePage (통계)
  10. Inquiries (문의 등록, 본인 문의 확인, 관리자 답변)
  11. CareerEvents (신청 일정 등록, 상태 관리, 캘린더 연동)

- **신청 일정 입력 기준**:
  - 장소명과 주소는 `장소 / 주소` 한 항목에 입력
  - 기존 장소·주소 분리 데이터는 수정 시 한 항목으로 합쳐 표시

- **문의 게시판 권한 기준**:
  - 로그인 사용자는 문의를 등록하고 본인 문의와 답변만 조회
  - 관리자 계정은 전체 문의를 조회하고 답변 작성·수정
  - 관리자 판별 UI에는 이메일 원문을 넣지 않고 해시 비교
  - 실제 데이터 접근은 Firestore Security Rules에서 인증 이메일로 제한
