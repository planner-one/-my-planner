### SC-03: AppContext + Firestore 동기화

- **목적**: 전역 상태와 Firestore 저장 전략 구현
- **주요 흐름**:
  1. `userService.ts`에서 사용자 데이터 로드/저장
  2. `AppContext.tsx`에서 Firestore 필드 전체 상태 관리
  3. `currentDataRef`로 최신 상태 추적
  4. 일반 변경은 1초 debounce 저장
  5. 레이아웃 저장과 로그아웃 전 저장은 즉시 저장
- **완료 기준**: 새로고침 후 데이터 복원, 계정 전환 시 localStorage 정리, pending save가 이전 uid와 계정 메타데이터를 유지

### SC-10: 태블릿/모바일 반응형

- **목적**: 태블릿과 모바일에서 대시보드와 기능 페이지를 사용할 수 있게 검토
- **검토 기준**:
  - 데스크톱, 태블릿 가로/세로, 모바일 폭별 주요 화면 확인
  - 모바일 대시보드 세로 카드형 위젯 구조
  - 하단 탭 대표 메뉴 6개
  - 모바일 상단 전체 메뉴를 통한 모든 페이지 접근
  - 하위 페이지 홈 복귀 버튼
  - 각 페이지 입력 폼, 카드 폭, 테이블/히트맵/캘린더 스크롤
  - 태블릿/모바일 위젯 드래그/리사이즈 지원 여부는 미확정
- **주의사항**:
  - 현재 완료 상태는 코드/빌드 기준입니다. 실제 로그인 세션에서 모바일 390px, 태블릿 세로 768px, 태블릿 가로 1024px 화면을 최종 확인하기 전에는 "실기기 최적화 완료"로 표현하지 않습니다.
  - 모바일 하단 탭은 대표 메뉴 6개만 노출합니다. 전체 페이지 접근은 상단 전체 메뉴 또는 메뉴 위젯으로 확인해야 합니다.
  - 캘린더, 습관 히트맵, 주간표, 출력 미리보기처럼 정보량이 많은 화면은 모바일에서 가로 스크롤 또는 축약 표시가 섞여 있으므로 텍스트 겹침, 버튼 잘림, 입력 폼 폭을 우선 확인합니다.
  - 태블릿/모바일 위젯 드래그/리사이즈는 지원 여부가 미확정입니다. 결정 전까지 지원 기능으로 안내하지 않습니다.
- **완료 메모**: 2026-06-29 기준 주요 페이지에 모바일/태블릿 브레이크포인트를 적용했고, 인앱 브라우저 연결 제한으로 실제 로그인 세션 육안 QA는 사용자 확인 권장 사항으로 남깁니다.
- **2026-07-16 실기기 피드백 1차 보강**: 특정 모바일 폭에서 일일 루틴 위젯이 content-box와 compact padding 사이를 반복하던 원인을 border-box 측정으로 수정했습니다. 모바일 전체 메뉴에 별도 계정/로그아웃 영역을 추가하고, 767px 이하 입력 16px·44px, textarea 96px, 본문 12px 여백, 하단 safe area 기준을 적용했습니다. 전체 모바일 화면의 개별 폼 최적화와 로그인 세션 육안 QA는 계속 진행합니다.

### SC-12: AI 링크 자동 정리 / 유료 기능 후보

- **목적**: 사용자가 링크를 입력하면 AI가 내용을 분석해 기회 일정, 내 신청, 지원 공고, 캘린더, 작업 목록, 메모 중 필요한 위치에 자동 정리하도록 설계
- **상태**: 2026-07-01 기준 1차 기반 구현. 실제 웹페이지 본문 분석, 링크 페이지 내부 이미지 자동 수집, 유료 권한, 사용량 제한은 추후 백엔드/API 연결 검토 대상으로 관리
- **구현된 범위**:
  - 전역 "링크 정리" 버튼과 링크 입력 모달
  - URL/사용자 메모 기반 초안 생성 서비스 경계
  - 이미지 포스터 자료 유형, 업로드 이미지 OCR, 포스터 텍스트 기반 날짜/시간/장소 초안 생성

  1. Google 로그인 후 `users/{uid}` 문서가 `null`인 신규 계정만 온보딩 v1 `pending` 상태로 자동 진입한다.
  2. 사용 목적 `일상 관리`, `업무·학습`, `취업 준비`, `기회·신청 관리` 중 하나를 선택한다.
  3. Todo는 오늘 끝낼 일, 작업은 날짜·우선순위가 있는 실행 항목, 프로젝트는 여러 작업을 묶은 결과물, 목표는 장기 목표와 단계·진행률이라는 역할을 안내한다.

  5. 목적별 추천 위젯을 미리 보고 `홈 적용하고 시작하기`로 첫 항목, 레이아웃, `completed` 상태를 한 번에 저장한다.
- **목적별 프리셋**:
  - 일상 관리: Todo, 캘린더, 일일 루틴, 빠른 메모, 하루 마무리

  - 취업 준비: 캘린더, Todo, 작업 흐름, 메뉴, 빠른 메모
  - 기회·신청 관리: 캘린더, Todo, 메뉴, 목표, 빠른 메모
- **저장/재진입 규칙**:
  - 프리셋은 기존 위젯 기본 크기를 사용해 48열에 순서대로 배치하고 ID는 `${widgetId}-starter-v1`로 고정한다.
  - 어느 단계에서든 건너뛰면 초안을 버리고, 빈 홈에만 범용 프리셋 `Todo·캘린더·빠른 메모·일일 루틴·메뉴`를 적용한다.
  - 기존 문서에 온보딩 필드가 없는 레거시 사용자는 자동 진입하지 않는다.
  - 완료·건너뛰기 저장 실패 시 화면을 유지하고 재시도하며, 다중 탭에서는 `completed > skipped > pending` 순위를 보존한다.
  - 프로필의 `사용 가이드 다시 보기`는 안내 전용으로 열고 데이터나 레이아웃을 바꾸지 않으며 마지막에 대시보드 편집 화면으로 이동한다.
- **완료 기준**:
  - 신규 문서 없음, 기존 문서 있음, 온보딩 필드 없는 레거시 문서의 자동 표시 여부가 각각 기대와 일치한다.
  - 네 목적별 첫 항목 타입·필수값·기본값과 추천 위젯의 순서, ID 중복, 48열 범위를 회귀 검사한다.
  - 완료, 건너뛰기, 저장 실패 후 재시도, 새로고침, 로그아웃·재로그인, 계정 전환, 다중 탭 병합을 검증한다.
  - 안내 전용 재열람이 기존 사용자 데이터와 홈 레이아웃을 변경하지 않는다.
  - 모바일 390px, 태블릿 세로 768px, 태블릿 가로 1024px, 데스크톱 로그인 세션에서 포커스·키보드·반응형을 확인한다.
