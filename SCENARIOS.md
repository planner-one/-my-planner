### SC-03: AppContext + Firestore 동기화

- **목적**: 전역 상태와 Firestore 저장 전략 구현
- **주요 흐름**:
  1. `userService.ts`에서 사용자 데이터 로드/저장
  2. `AppContext.tsx`에서 Firestore 필드 전체 상태 관리
  3. `currentDataRef`로 최신 상태 추적
  4. 일반 변경은 1초 debounce 저장
  5. 레이아웃 저장과 로그아웃 전 저장은 즉시 저장
- **완료 기준**: 새로고침 후 데이터 복원, 계정 전환 시 localStorage 정리, pending save가 이전 uid와 계정 메타데이터를 유지

### SC-04: 앱 라우팅 + 네비게이션

- **목적**: `RouterContext` 기반 페이지 전환, 사이드바/하단 탭/메뉴 위젯 연결
- **현재 page id**:
  - `dashboard`, `calendar`, `tasks`, `todos`, `habits`, `goals`, `projects`, `career`
  - `weekly`, `daily`, `notes`, `journal`, `profile`, `inquiries`, `print`
- **완료 기준**: 사이드바/모바일 전체 메뉴/메뉴 위젯의 모든 페이지가 실제 컴포넌트로 연결되고, 하위 페이지에서 홈 복귀 버튼이 제공됨. 데스크톱 사이드바는 접힘 레일과 펼침 메뉴를 제공하며 페이지 이동 후에도 사용자가 선택한 상태를 유지함

### SC-07: 위젯 구현/검토

- **목적**: 대시보드 위젯 14종 구현과 리사이즈 반응형 검토
- **현재 등록 위젯**:
  1. 현재 시각
  2. 빠른 메모
  3. 캘린더
  4. 오늘 할 일
  5. 일일 루틴
  6. 집중 타이머
  7. 카운터
  8. 생산성 추이
  9. 목표
  10. 메뉴
  11. 하루 마무리
  12. 예정된 작업
  13. 날씨
  14. 저널 알림
- **정리된 위젯**:
  - 최우선 목표: 목표 위젯의 오늘 집중 섹션으로 통합
  - 빠른 추가: 오늘 할 일 위젯과 중복되어 삭제
- **완료 기준**: 각 위젯이 AppContext 데이터와 연동되고 크기별 주요 UI가 잘리지 않으며, 위젯 등록 순서와 모바일 높이 정의가 문서와 일치

### SC-10: 태블릿/모바일 반응형

- **목적**: 태블릿과 모바일에서 대시보드와 기능 페이지를 사용할 수 있게 검토
- **검토 기준**:
  - 데스크톱, 태블릿 가로/세로, 모바일 폭별 주요 화면 확인
  - 모바일 대시보드 세로 카드형 위젯 구조
  - 하단 탭 대표 메뉴 6개
  - 모바일 상단 전체 메뉴를 통한 모든 페이지 접근
  - 하위 페이지 홈 복귀 버튼
  - 각 페이지 입력 폼, 카드 폭, 테이블/히트맵/캘린더 스크롤
  - 태블릿/모바일 위젯 드래그/리사이즈 지원 여부는 미확정
- **완료 메모**: 2026-06-29 기준 주요 페이지에 모바일/태블릿 브레이크포인트를 적용했고, 인앱 브라우저 연결 제한으로 실제 로그인 세션 육안 QA는 사용자 확인 권장 사항으로 남깁니다.
