# my_planner

프론트엔드 + Firebase만으로 운영되는 플래너 서비스 (백엔드 없음).

## 기술 스택
- Frontend: (프롬프트에서 확정 예정)
- Database / Auth / Hosting: Firebase

## 문서 구조
- [REQUIREMENTS.md](REQUIREMENTS.md) — 요구사항 목록
- [SCENARIOS.md](SCENARIOS.md) — 시나리오 (기능 흐름)
- [PROGRESS.md](PROGRESS.md) — 작업 진행 상태

## GitHub
https://github.com/planner-one/-my-planner.git

## 작업 방식
1. 프롬프트 수신 → 요구사항 분석 → REQUIREMENTS.md 업데이트
2. 시나리오 도출 → SCENARIOS.md 업데이트
3. 시나리오 순서대로 구현 → PROGRESS.md 업데이트
