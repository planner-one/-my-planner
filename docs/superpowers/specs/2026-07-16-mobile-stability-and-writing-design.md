# 모바일 안정화·글쓰기 1차 개선 설계

_작성일: 2026-07-16_

## 목표

모바일 대시보드에서 일일 루틴 위젯이 반복해서 떨리는 문제를 제거하고, 모바일 전체 메뉴에 다른 페이지 항목과 분리된 로그아웃 버튼을 제공한다. 동시에 모바일 글쓰기의 첫 개선으로 입력 글자 크기, 터치 높이, 화면 좌우 여백, 하단 안전 영역을 공통 보정한다.

## 확인된 원인

`useWidgetSize`는 위젯 루트의 `contentRect`를 측정한다. `HabitWidget`은 측정 폭이 310px 미만이면 좌우 패딩을 18px에서 12px로 바꾼다. 외부 폭이 334~345px이면 패딩 변경으로 content-box 폭이 임계값 양쪽을 오가며 `compact`가 계속 전환된다.

- 외부 폭 340px, 일반 패딩: 측정 폭 304px → compact 전환
- 외부 폭 340px, compact 패딩: 측정 폭 316px → 일반 전환
- 위 두 상태가 반복되어 행 간격·체크박스 크기·패딩이 계속 바뀐다.

## 검토한 접근

1. `HabitWidget`의 임계값이나 패딩만 바꾸기: 가장 작지만 다른 폭이나 다른 위젯에서 같은 문제가 다시 생길 수 있다.
2. `useWidgetSize`가 border-box를 측정하도록 수정: 자기 패딩에 영향받지 않는 외부 크기로 반응형 상태를 결정하므로 근본 원인을 공통 경계에서 제거한다. 이 방식을 채택한다.
3. 모바일에서 위젯 크기 적응을 모두 끄기: 떨림은 사라지지만 작은 카드 대응력이 떨어져 채택하지 않는다.

## 화면 설계

### 독립 로그아웃

모바일 `전체 메뉴`의 페이지 그리드 아래에 구분선을 둔 계정 영역을 추가한다. 사용자 아바타·이름과 전체 폭 `로그아웃` 버튼을 배치한다. 로그아웃 버튼은 페이지 이동 항목에 섞지 않으며, 저장과 로그아웃이 진행되는 동안 중복 입력을 막고 진행 문구를 표시한다. 데스크톱 사이드바의 기존 로그아웃 흐름과 같은 `saveNow → signOut` 순서를 재사용한다.

### 글쓰기 1차 최적화

모바일 767px 이하에서 다음 공통 기준을 적용한다.

- 본문 좌우 여백: 20px에서 12px로 축소해 입력 공간 확보
- 텍스트 입력·선택 컨트롤: 최소 높이 44px
- `input`, `select`, `textarea`: 16px 글자 크기로 모바일 브라우저 자동 확대와 작은 글씨 입력 불편 완화
- `textarea`: 최소 높이 96px와 읽기 쉬운 줄 높이 적용
- 하단 탭: 44px 터치 영역과 `safe-area-inset-bottom` 반영
- 스크롤 본문: 하단 탭과 키보드 주변에서 마지막 입력 영역이 가려지지 않도록 하단 scroll padding 반영

체크박스·라디오·색상·범위 입력은 공통 44px 입력 규칙에서 제외해 기존 전용 UI를 유지한다.

## 데이터와 오류 처리

Firestore 문서 구조와 저장 payload는 변경하지 않는다. 로그아웃은 기존처럼 최신 사용자 데이터를 먼저 즉시 저장한다. 저장 또는 인증 로그아웃이 실패하면 버튼을 다시 활성화하고 현재 화면을 유지한다.

## 검증

- 334px, 340px, 345px 외부 폭에서 compact 판정이 한 상태로 유지되는 회귀 검사
- border-box 관찰, 동일 크기 state 갱신 방지 계약 검사
- 모바일 전체 메뉴의 독립 로그아웃과 저장 후 로그아웃 연결 검사
- 모바일 공통 입력·본문 여백·터치 영역·안전 영역 CSS 계약 검사
- 타입 검사, 전체 기존 회귀 검사, 프로덕션 빌드
- 로컬 390px 브라우저에서 로그인 전 화면 로딩·오류 오버레이 확인. 로그인 후 데이터 화면은 자동 세션이 없으면 코드·회귀 검사와 실제 배포 후 사용자 확인으로 구분한다.
