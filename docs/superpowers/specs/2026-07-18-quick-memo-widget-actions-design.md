# 빠른 메모 위젯 편집·보관 UX 설계

_작성일: 2026-07-18_

## 목표

빠른 메모 위젯의 스크롤바를 평소에는 숨기고 실제 스크롤 중에만 표시한다. 상단 `보관함` 왼쪽의 연필 버튼으로 편집 모드를 열고, 각 메모는 항상 표시되는 `×`로 보관하거나 편집 모드에서 수정·완전 삭제할 수 있게 한다.

## 화면과 상호작용

- 상단 액션은 `연필 → 보관함` 순서로 배치한다.
- 연필 버튼에는 `빠른 메모 편집` 접근성 이름과 툴팁을 제공한다.
- 연필을 누르면 편집 모드가 열리고 상단 아이콘은 체크로 바뀐다. 체크를 누르면 편집 모드를 종료한다.
- 평소 각 메모 오른쪽에는 `×`만 표시한다. `×`는 메모를 완전히 삭제하지 않고 `archivedAt`을 기록해 보관함으로 이동한다.
- 편집 모드에서는 각 메모 오른쪽에 `수정`, `삭제`, `×`를 함께 표시한다.
- `수정`을 누르면 본문이 인라인 입력으로 바뀐다. Enter 또는 포커스 이탈은 저장하고, Shift+Enter는 줄바꿈, Escape는 취소한다.
- 빈 수정 내용은 저장하지 않고 기존 내용을 유지한다.
- `삭제`는 확인창을 거친 뒤에만 메모를 완전히 제거한다.
- 작성 시각은 본문 아래에 작게 두어 오른쪽 액션과 충돌하지 않게 한다.

## 상태와 데이터

`MemoActions`와 `MemoWidget`은 대시보드 카드의 서로 다른 위치에서 렌더링된다. 두 컴포넌트의 편집 모드는 `MemoWidget.tsx` 모듈 내부의 작은 `useSyncExternalStore` 저장소로 공유한다. 대시보드 편집기는 같은 종류의 위젯을 한 개만 유지하므로 인스턴스별 영속 상태는 추가하지 않는다.

메모 수정은 기존 `QuickMemoEntry.content`와 `updatedAt`만 갱신한다. 보관은 `archivedAt`과 `updatedAt`을 갱신하고, 완전 삭제만 배열에서 항목을 제거한다. Firestore 문서 구조와 자동 저장 흐름은 변경하지 않는다.

## 스크롤바

목록은 기존처럼 세로 스크롤을 유지한다. 스크롤바 폭과 여백은 고정하되 thumb와 track을 투명하게 두어 레이아웃 이동을 막는다. `scroll` 이벤트가 발생하면 `is-scrolling` 클래스를 잠시 적용해 thumb만 표시하고, 마지막 스크롤 뒤 약 700ms 후 다시 숨긴다.

## 접근성과 오류 방지

- 아이콘 버튼에는 `title`, `aria-label`, `aria-pressed`를 필요에 맞게 제공한다.
- 보관과 완전 삭제를 각각 `보관함으로 이동`, `완전히 삭제`로 명확히 구분한다.
- 완전 삭제에는 기존 노트 페이지와 같은 확인 문구를 사용한다.
- 편집 중 보관·삭제된 항목은 로컬 편집 상태도 함께 정리한다.
- 스크롤 타이머는 컴포넌트가 사라질 때 해제한다.

- 스크롤 중 표시와 지연 숨김 CSS·이벤트 계약 검사
- 기존 빠른 메모 추가와 노트 페이지 보관함 데이터 호환성 확인
