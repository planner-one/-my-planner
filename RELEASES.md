# 릴리즈 노트

_마지막 업데이트: 2026-07-01_

플래너는 기능 범위와 안정화 단계를 기준으로 버전을 나눕니다.

## 버전 기준

| 버전 | 이름 | 기준 |
|------|------|------|
| v0.1.x | Foundation | 인증, Firebase 저장, 대시보드, 기본 위젯 중심의 초기 플래너 |
| v0.2.x | Core Planner | 사이드바 기능 페이지, 캘린더/기회 일정/Todo/일일·주간 흐름이 연결된 사용 가능 버전 |
| v0.3.x | UX Stabilization | 실제 사용 화면 기준의 접근성, 모바일/태블릿 QA, 반복 작업 UX 안정화 |
| v0.4.x | Integration Ready | 추후 백엔드/API 연결을 고려한 서비스 경계와 외부 연동 준비 |
| v0.5.x | Paid Intelligence Candidate | 링크 자동 분석, 유료 기능, 사용량 제한, 약관/개인정보 흐름 검토 |

## v0.2.0 - Core Planner

_릴리즈일: 2026-06-30_

핵심 변경:

- 데스크톱 사이드바를 접힘 레일/펼침 상태 유지형으로 개선
- Todo 기록의 체크 안 된 항목을 오늘 할 일로 다시 올리는 흐름 보강
- 기회 일정, 캘린더, 일일/주간 플래너의 날짜 기반 연결 강화
- 캘린더 관리 페이지와 기회 일정 효율화
- 프로젝트, 주간, 일일, 저널, 프로필, 플래너 출력 페이지 구현/보강
- 태블릿/모바일 반응형 코드 기준 정리
- 프로필 화면에 현재 앱 버전과 릴리즈 요약 표시

## v0.3.0 - Workflow Visibility

_릴리즈일: 2026-07-01_

핵심 변경:

- 대시보드에 작업 관리, 프로젝트, 목표를 함께 보여주는 작업 흐름 위젯 추가
- 작업 관리, 프로젝트, 목표 페이지의 역할 구분과 요약 지표 강화
- 일일 플래너에 오늘 마감인 작업, 프로젝트, 목표를 연결 표시
- 주간 플래너에 작업 관리 마감까지 포함해 주간 흐름 표시
- 정식 노트 페이지, 링크 정리, 이미지 포스터 OCR 기반 기능을 v0.3 기준으로 정리

## v0.3.1 - Page UX Stabilization

_릴리즈일: 2026-07-01_

핵심 변경:

- 작업 관리 페이지에 실행 대기열, 빠른 완료, 상태 균형 보드를 추가
- 프로젝트 페이지에 다음 결과물, 프로젝트 흐름, 하위 작업 스냅샷을 추가
- 목표 관리 페이지에 오늘 집중 달성률과 다음 장기 목표 보드를 추가
- 일일 플래너에 오늘 진행감, 다음 Todo, 다음 일정, 연결 마감 브리핑을 추가
- 주간 플래너에 이번 주 운영, 첫 마감, 가장 바쁜 날 요약을 추가
- 저널 페이지에 기록 상태, 빠른 작성 시작, 최근 에너지 흐름을 추가

## v0.3.2 - Application Tracking

_릴리즈일: 2026-07-01_

핵심 변경:

- 저축계좌, 내일두배통장, 기관 멘토링 같은 제도성 신청을 관리하는 내 신청 페이지 추가
- 사람인, 잡플래닛, 원티드, 점핏, 그룹바이 등 채용 공고를 모으는 지원 공고 페이지 추가
- 지원 공고 링크에서 플랫폼을 감지하고, 이미지 OCR 텍스트/붙여넣은 텍스트에서 초안 보정
- 내 신청과 지원 공고 데이터를 기존 기회 일정과 분리해 `personalApplications`, `jobPostings`로 저장
- 내 신청/지원 공고의 마감일, 신청일/지원일, 결과일, 시작/종료일을 캘린더에 연결
- 페이지 단위 코드 분리와 React/Firebase/그리드/차트/OCR vendor 청크 분리로 500KB 초과 번들 경고 제거

## v0.3.3 - Job Link Drafts

_릴리즈일: 2026-07-01_

핵심 변경:

- 지원 공고 추가 폼 위에 접힌 "링크로 초안 만들기" 보조 패널 추가
- 공고 링크를 넣으면 플랫폼, 원본 링크, 포지션/마감/키워드 초안을 입력 폼에 자동 반영
- 저장된 지원 공고 카드에서도 링크 행의 "분석 반영"으로 플랫폼/마감/키워드를 보정
- 실제 웹페이지 본문 자동 분석과 링크 내부 이미지 수집은 추후 프리미엄 후보로 안내

## v0.3.4 - Company Job Links

_릴리즈일: 2026-07-01_

핵심 변경:

- 기업 자체 채용 페이지, Google Sites 기업 상세, 인크루트 기업 도메인을 지원 공고 링크로 인식
- 링크만 확보한 공고도 `기업 미정`, `공고 확인 필요` 상태로 저장 가능
- 브라우저에서 본문을 읽을 수 있는 링크는 기업명/직무/키워드 초안에 반영
- CORS, 로그인, 리다이렉트로 본문을 읽지 못하는 링크는 URL·도메인·OCR/붙여넣기 텍스트 기반으로 보정

## v0.3.5 - Job Content Parsing

_릴리즈일: 2026-07-01_

핵심 변경:

- 링크 초안 패널 안에 페이지 내용 붙여넣기 영역 추가
- 붙여넣은 공고 내용에서 기업명, 채용 직무/분야, 소재지, 고용형태를 폼에 자동 반영
- 기술스택, 연봉/급여, 지원 자격, 우대사항은 키워드와 메모로 구조화
- Google Sites 경로명은 실제 기업명과 다를 수 있어 본문 없이 회사명으로 확정하지 않도록 보정

## v0.3.6 - Job Draft Accuracy

_릴리즈일: 2026-07-01_

핵심 변경:

- Google Sites 자동 본문이 실제 공고 내용인지 확인하고, 공고 힌트가 없으면 분석에서 제외
- `nextsystem` 같은 경로명을 회사명으로 잘못 채우지 않도록 보정
- 지원 공고 입력 폼의 붙여넣은 내용 영역에 즉시 반영 버튼 추가
- 소재지, 사업내용, 고용형태, 연봉 같은 라벨 파싱 오류 수정

## v0.3.7 - Job Draft Verification

_릴리즈일: 2026-07-01_

핵심 변경:

- 지원 공고 링크/본문 분석 로직을 `src/utils/jobPostingDraft.ts` 공통 유틸로 분리하고 화면 중복 파서를 제거
- Google Sites 링크만 있는 경우 `nextsystem` 같은 경로명을 기업명으로 확정하지 않도록 자동 테스트 추가
- Google Sites 공고 본문을 붙여넣으면 기업명, 채용 직무/분야, 소재지, 고용형태, 기술스택을 추출하는 자동 테스트 추가
- 인크루트 기업 도메인 링크에서 `KDB`처럼 서브도메인 기반 기업명을 잡는 회귀 테스트 추가
- `기업 미정`, `공고 확인 필요` placeholder가 이후 붙여넣은 본문 재반영을 막던 문제 수정
- Google Sites meta 제목의 `... - 주식회사 스탁키퍼` 형식에서 실제 회사명만 추출하도록 보강
