## v0.3.12 UI System Pass

목표: 기능은 많은데 화면이 복잡해지는 문제를 줄입니다.

- 공통 카드/패널/입력/버튼 스타일 기준 정리
- 대시보드 위젯 카드 반경과 페이지 카드 반경 통일 검토
- 오늘 할 일, 지원 공고, 내 신청, 캘린더처럼 정보량 많은 화면의 밀도 조정
- 모바일 390px, 태블릿 세로 768px, 태블릿 가로 1024px 실제 화면 체크
- 긴 텍스트, 버튼, 날짜 입력, 셀렉트가 잘리지 않는지 확인
- 사이드바/모바일 전체 메뉴의 진입 우선순위 재검토

## v0.3.13 Planner Flow Clarity

목표: 사용자가 "어디에 뭘 적어야 하는지" 헷갈리지 않게 합니다.

- Todo: 오늘 끝낼 구체 작업
- 작업 관리: 날짜/우선순위가 있는 개별 실행 작업
- 프로젝트: 여러 작업을 묶은 결과물
- 목표: 장기 목표와 단계/진행률
- 오늘 방향: 오늘 하루만 붙잡을 기준
- 기회 일정: 외부 프로그램/행사/교육/공모전
- 내 신청: 이미 신청했거나 상태 추적할 제도성 신청
- 지원 공고: 채용 공고 링크와 지원 상태

후보 작업:

- 각 페이지 상단 설명을 더 짧고 명확하게 정리
- 대시보드 작업 흐름 위젯에 "오늘 해야 할 것 / 이번 주 볼 것 / 장기 진행" 구분 추가
- 캘린더에서 날짜 항목을 "해야 할 일 / 신청·지원 / 마감 / 기록"으로 묶어 보기
- 목표 페이지의 오늘 방향 히스토리 표시 여부 검토

## v0.3.14 Data Quality & AI Readiness

목표: 추후 AI 맞춤 기능에 쓸 수 있는 데이터를 깨끗하게 쌓습니다.

- 상태 변경 이력 저장이 필요한 데이터 확정: 내 신청, 지원 공고, 프로젝트, 목표
- `sourceUrl`, `keywords`, `createdAt`, `updatedAt` 필드 일관성 점검
- 노트/지원 공고/내 신청/기회 일정 통합 검색 후보 설계
- 사용자 데이터 내보내기 JSON 기능 후보 설계
- 사용자 데이터 삭제/초기화 정책 후보 설계
- 개인정보/원본 링크 보존 범위와 사용자 동의 문구 설계

## v0.4.0 Integration Ready

목표: 프론트엔드 단독으로 어려운 기능을 서버 경계로 옮길 준비를 합니다.

- `/api/job-posting-page`를 Firebase Functions 또는 별도 백엔드 API로 이관
- `/api/job-posting-image` 이미지 프록시/OCR 후보를 운영 환경에서 사용할 수 있게 설계
- 외부 페이지 접근 실패, robots/약관, 로그인 필요한 페이지 처리 정책 정리
- API 키와 AI 모델 호출을 클라이언트에 노출하지 않는 구조 설계
- 무료/유료 권한 체크가 가능한 데이터 모델 후보 설계

## v0.5.0 Paid Intelligence Candidate

목표: 유료 기능 후보를 실제 서비스 기능으로 만들기 전 필요한 정책과 UX를 갖춥니다.

- 링크 자동 분석 사용량 제한
- AI 추출 결과 미리보기/수정/확인 단계 고정
- 이미지 포스터 자동 분석과 후보 이미지 선택
- 지원 공고 맞춤 준비 체크리스트
- 내 신청/지원 공고/기회 일정 기반 알림 추천
- 사용자별 패턴 분석: 미완료 Todo, 마감 임박, 자주 쓰는 키워드, 지원 상태 흐름
- 결제/권한/환불/오류 처리 정책 초안

## 디자인 개선 백로그

| 우선순위 | 항목 | 이유 |
|---|---|---|
| P0 | 폼이 많은 페이지의 입력 줄 정리 | 지원 공고/내 신청/기회 일정은 정보량이 많아 스캔성이 중요 |
| P0 | 모바일 긴 텍스트와 버튼 줄바꿈 QA | 실제 사용에서 잘림/겹침 가능성이 큼 |

| P1 | 대시보드 위젯 기본 배치 프리셋 | 새 사용자나 초기화 사용자의 시작 경험 개선 |
| P1 | 캘린더 항목 색상/범례 정리 | 일정 종류가 늘어 구분 피로도가 높아질 수 있음 |
| P2 | 버전 업데이트 안내 배너 | 사용자가 무엇이 바뀌었는지 알기 쉽게 하기 위함 |
